\begin{abstract}
Humanoid robots must decide whether a user utterance requires a verbal reply
(\texttt{chat}) or a physical action (\texttt{motion\_query}) before the
user finishes speaking.
We study this binary intent task on a 3{,}000-utterance dataset and compare
four text encoders (TF-IDF, CLIP, Qwen3-Embedding-0.6B, MiniLM-L6-v2)
with two lightweight classifier heads (a linear LogReg-style head and a small
MLP).
On full sentences, the eight encoder--head combinations reach 98.3--99.7\%
test accuracy; CLIP+MLP performs best on the standard test split while
TF-IDF+LogReg remains a strong compact baseline.
On an expanded 150-example hard test set, however, accuracy drops sharply:
Qwen3+MLP performs best at 84.0\%, while TF-IDF+LogReg falls to 78.7\%.
For prefix-based early prediction at confidence threshold~0.7, TF-IDF+LogReg
achieves 99.2\% accuracy among first non-wait decisions, covers 86.7\% of test
utterances, and commits after a mean 42.1\% of the sentence (median 33.3\%).
We recommend TF-IDF+LogReg as the simplest default and use the higher-coverage
embedding models when early-decision coverage is more important.
\end{abstract}

\section{Introduction}

In spoken interaction with humanoid robots, not every user query should trigger
the same response modality.
Informational questions (\emph{``What is machine learning?''}) call for
text generation, whereas imperative or demonstrative requests
(\emph{``Wave your hand''}) should activate motion planning.
Misclassifying \texttt{chat} as \texttt{motion\_query} causes unnecessary or
embarrassing robot gestures (\emph{false motion}); the reverse misses motion
requests (\emph{false text}).

Streaming speech adds latency: waiting for a complete sentence delays motion
preparation.
We therefore evaluate both \textbf{full-sentence classification} and
\textbf{prefix-based early prediction}, where the model sees only the first
20--100\% of words.

Our contributions are:
\begin{itemize}
  \item A balanced 3{,}000-utterance dataset for \texttt{chat} vs.\
        \texttt{motion\_query}, with a 150-example hard test set of boundary
        cases.
  \item A controlled $4 \times 2$ comparison (encoder $\times$ head) reporting
        accuracy, macro-F1, error counts, and tracked training runtime.
  \item Prefix evaluation simulating incremental speech, measuring accuracy at
        five prefix ratios and early-decision behavior under confidence
        thresholds.
\end{itemize}

\subsection{Related Work}
Intent classification for spoken dialogue systems has a long history, from
keyword spotting and SVM-based slot filling to modern transformer encoders
fine-tuned on task-specific corpora~\cite{zhang2021intent}.
In human--robot interaction, prior work often treats motion commands as a
separate skill invocation layer rather than a first-class routing decision
made \emph{before} response generation.
Our setting differs in two ways: (i)~we target a lightweight binary router
suitable for on-board deployment, and (ii)~we explicitly evaluate
\emph{partial} utterances to mimic streaming ASR, following the spirit of
early-exit and incremental decoding methods used to reduce dialogue latency.

\subsection{Problem Setting}
Figure~\ref{fig:pipeline} summarizes the intended deployment loop.
An ASR stream feeds word prefixes into the classifier; when confidence exceeds
a threshold~$\tau$, the robot either prepares a motion pipeline or continues
standard dialogue generation.
Low-confidence predictions defer the decision until more speech arrives.

\section{Method}

\subsection{Task and Labels}
Each utterance is labeled \texttt{chat} (verbal response) or
\texttt{motion\_query} (physical action).
We use macro-F1, false-motion rate (FMR), and false-text rate (FTR) alongside
accuracy.

\subsection{Dataset}
The full-input dataset used in the tracked experiments contains 3{,}000
DeepSeek-generated utterances (1{,}500 per class).
We split 80/10/10 into train/val/test (2{,}400 / 300 / 300) with
stratification (seed~20260518).
A separate \emph{hard test} set of 150 manually curated boundary utterances
(73 \texttt{chat}, 77 \texttt{motion\_query}) evaluates robustness. It includes
chat examples with motion vocabulary, negation, capability questions,
hypotheticals, and word-only requests, plus motion examples with indirect
requests, demonstrations, negation traps, and context-driven physical actions.

Data were collected in two stages for the current experiments.
First, 500 hand-written seed utterances established label conventions and
boundary patterns.
Second, DeepSeek-generated batches expanded coverage with LLM diversity and
formed the balanced 3{,}000-row dataset used for V1 and as the source data for
V2 prefix generation.
Table~\ref{tab:examples} illustrates typical cases and the ambiguity targeted
by the hard test.

\subsection{Encoders and Classifier Heads}
We fix the encoder and vary only the classification head:
\begin{itemize}
  \item \textbf{TF-IDF}: bag-of-words with 1--2-gram features (max 10{,}000
        features for V1 and 15{,}000 for V2).
  \item \textbf{CLIP}: \texttt{sentence-transformers/clip-ViT-B-32} text encoder.
  \item \textbf{Qwen3}: \texttt{Qwen/Qwen3-Embedding-0.6B}.
  \item \textbf{MiniLM}: \texttt{sentence-transformers/all-MiniLM-L6-v2}.
\end{itemize}
Heads: (1)~\textbf{LogReg}---a single PyTorch linear layer trained as a
logistic-regression-style classifier on cached embeddings or TF-IDF vectors;
(2)~\textbf{MLP}---Linear(256) $\to$ ReLU $\to$ Dropout $\to$ Linear, trained
with AdamW.

Embedding vectors are pre-computed once per version and encoder, then reused
for both heads of the same encoder.

\subsection{Prefix (Early) Evaluation}
From each test utterance we extract word-level prefixes at ratios
20\%, 40\%, 60\%, 80\%, and 100\%, yielding 1{,}500 test prefix rows from
300 source utterances.
Models classify each prefix independently with labels
\{\texttt{chat}, \texttt{motion\_query}, \textsc{wait}\}.
We also measure \emph{early decision}: for confidence threshold~$\tau$,
the model commits at the earliest non-wait prefix whose predicted confidence
exceeds $\tau$. We report first-decision accuracy, decision coverage, and the
visible prefix ratio where the first decision is made; \textsc{wait} is treated
as abstention rather than as a correct final answer.

\subsection{Hard-Test Construction}
The hard test is not used for training or model selection. It is a held-out
diagnostic set designed to stress the lexical shortcuts that make the standard
random split easy: motion words inside chat questions, negated commands, and
indirect physical requests without simple imperative phrasing.

\subsection{Implementation Details}
All heads use cross-entropy loss and AdamW ($\text{lr}=10^{-3}$, weight decay
$10^{-4}$). V1 uses batch size~64; V2 uses batch size~128.
Training runs for at most 60 epochs with early stopping patience~15 on
validation loss.
Embedding caches are built once per encoder; both LogReg and MLP heads then
train on the cached vectors, isolating encoder quality from head capacity.
All splits use seed~20260518. The tracked summaries report classifier-head
training/runtime after feature extraction, not a separate single-utterance
latency benchmark.

On the 150-example hard set (Table~\ref{tab:hard}), the standard-test ceiling
largely disappears. Qwen3+MLP performs best at 84.0\%, followed by CLIP+MLP at
83.3\%. TF-IDF+LogReg drops from 99.3\% on the standard test split to 78.7\%,
which suggests that the original split is strongly lexically separable.
The dominant failure mode is false motion: chat utterances that mention actions
are often routed to \texttt{motion\_query}.

\subsection{Error Analysis}
Hard-set failures cluster into three patterns.
\textbf{(1)~Lexical overlap:} chat utterances that mention motion vocabulary
(\emph{``Can you explain how to wave properly?''}) are misclassified as
\texttt{motion\_query} by many models.
\textbf{(2)~Negation:} \emph{``Please do not move, just explain the answer''}
can still trigger motion because the classifier attends to \emph{move} despite
the prohibitive context.
\textbf{(3)~Capability queries:} \emph{``Can you show me what you can do?''}
is ambiguous between a verbal capability summary and a physical demonstration;
TF-IDF and frozen embedding models both struggle without task-specific examples
that separate explanation from execution.
The new hard-test results therefore support the hypothesis that high
in-distribution accuracy mainly reflects surface lexical separability rather
than robust semantic grounding.

The three-class prefix task is harder than full-sentence classification because
the model must also learn when to abstain with \textsc{wait}. Accuracy is very
high at 20\%, where many examples are correctly labeled as \textsc{wait}, then
dips around 60--80\% as more examples become decision-ready before recovering
near full context.

At confidence threshold $\tau = 0.7$ (Table~\ref{tab:early}), TF-IDF+LogReg
is extremely accurate when it decides (99.2\%) but covers 86.7\% of test
utterances. CLIP+MLP and Qwen3+LogReg reach higher correct-decision coverage
(96.7\%) while maintaining roughly 99\% first-decision accuracy.
Raising~$\tau$ reduces wrong non-wait decisions but increases abstention; for
TF-IDF+LogReg, $\tau=0.9$ reaches 100\% row-level decision accuracy with lower
first-decision coverage (71.7\%).

\subsection{Deployment Recommendations}
For a resource-constrained humanoid, we suggest using TF-IDF+LogReg as the
default lightweight router and committing when confidence $\ge 0.7$.
If higher early-decision coverage is more important than simplicity, CLIP+MLP
or Qwen3+LogReg provide stronger coverage on the current prefix benchmark.

\section{Conclusion}

We systematically compared four text encoders and two classifier heads for
humanoid intent routing on 3{,}000 utterances.
Full-sentence accuracy is high across all models, but hard boundary cases and
prefix evaluation expose meaningful differences.
\textbf{TF-IDF+LogReg} is the simplest strong default: 99.3\% full-sentence
test accuracy and 99.2\% first-decision accuracy at 86.7\% coverage under the
V2 thresholded prefix setting.
\textbf{CLIP+MLP} gives the highest full-sentence accuracy (99.7\%) and strong
early-decision coverage, while \textbf{Qwen3+LogReg} offers the best
first-decision accuracy among the high-coverage prefix models.
However, the 150-example hard test shows that standard random-split accuracy
overstates robustness: the best hard-test model is Qwen3+MLP at 84.0\%, and
TF-IDF+LogReg falls to 78.7\%.

Limitations include the modest 3{,}000-row generated dataset, English-only
coverage, the need to expand the hard test with more natural ASR-style
utterances, and prefix simulation by word truncation rather than real ASR
output.
Future work will integrate streaming speech recognition and deploy the chosen
model on a physical humanoid platform with confidence-gated motion preparation.
